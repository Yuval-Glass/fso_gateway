FSO Gateway POC - High-Performance Packet Recovery SystemOverviewThe FSO Gateway is a specialized bridge designed to connect standard Ethernet networks over Free Space Optical (FSO) links. FSO communication is highly susceptible to atmospheric turbulence, which causes deep fading and significant "burst" packet losses.This system mitigates these issues by implementing a robust pipeline of Fragmentation, Matrix Interleaving, and Forward Error Correction (FEC) using the Wirehair (Fountain Code) library.Project Status: March 2026Current Milestone: Phase 5 (Interleaving & Burst Resilience) - COMPLETED.The system has been rigorously stress-tested and proven to recover data with 100% accuracy under severe burst-loss conditions.Completed Core Components1. Infrastructure (Phases 0-1)Linux Environment: Isolated development environment.Modern Build System: Makefile supporting C/C++ with automatic dependency tracking.Advanced Logging: Multi-level logging system with CLI configuration.Memory Safety: Full integration with AddressSanitizer (ASan) for debugging.2. Packet Processing & Fragmentation (Phase 2)Implements logic to break down large Ethernet frames (including 9000B Jumbo Frames) into fixed-size symbols.Ensures bit-exact reassembly at the receiver side.3. Block Management & FEC Wrapper (Phases 3-4)Block Builder: Groups symbols into blocks of size $K$. Includes timeout logic for flushing partial blocks.Wirehair Integration: A professional C wrapper for the Wirehair library.The "Cold Buffer" Fix: Resolved a critical mathematical bug in GF(256) by implementing full-reconstruction semantics and strictly aligned memory buffers.4. Matrix Interleaving & Burst Resilience (Phase 5)Matrix Interleaver: Disperses symbols from the same FEC block across the time domain.Sparse Deinterleaver: Assembles blocks from an interleaved stream, capable of handling "holes" (lost packets) without state corruption.End-of-Stream Flushing: Implemented deinterleaver_tick to ensure final partial blocks are recovered even if the stream ends abruptly.Performance Verification (The Task 19 Stress Test)The system was subjected to an automated stress test suite (Sweep Test) to define its theoretical limits:Burst Length Sweep: Verified 100% recovery for burst lengths up to 3,200 consecutive packets (with $D=100, M=32$).Burst Start Sweep: Proven resilience regardless of where the interference occurs in the transmission cycle.Exact Boundary Testing: Mathematically verified that the system recovers up to exactly $M$ losses per block and fails gracefully at $M+1$.Result: 20/20 Success Rate across all automated test cases.Current Architecture (The Pipeline)Capture: Ingests Raw Ethernet Packets (currently simulated, moving to Raw Sockets).Fragmenter: Slices data into fixed symbols.FEC Encoder: Generates $M$ repair symbols for every $K$ source symbols.Interleaver: Shuffles symbols into a column-major matrix for transmission.Channel (Simulation): Injects deterministic or random burst losses.Deinterleaver: Re-orders symbols and identifies missing indices.FEC Decoder: Uses repair symbols to reconstruct missing source data.Reassembler: Restores the original Ethernet frame.Roadmap -
